Use Git and GitHub CLI with an Existing Unity Project

You already have a Unity project — let’s connect it to Git and GitHub step by step using the GitHub CLI.

  • 1. Open your Unity project in VS Code

    Launch VS Code and open your Unity project folder (File → Open Folder). You should see folders like Assets and ProjectSettings.

    Open the terminal with Ctrl + `.

  • 2. Verify you are inside the Unity folder

    Make sure you are in the Unity project root (the same level as Assets/).

    ls
  • 3. Initialize Git in your project

    Set up Git version control for this folder.

    git init
  • 4. Create a Unity .gitignore file

    This excludes build folders and temporary Unity files from your commits.

    curl -o .gitignore https://raw.githubusercontent.com/github/gitignore/main/Unity.gitignore
  • 5. (Optional) Add a README file

    This is a short description of your project that appears on GitHub.

    echo "# My Unity Project" > README.md
  • 6. Stage and commit your project files

    This records your current project state in Git.

    git add . git commit -m "Initial commit - Unity project setup"
  • 7. Log in to GitHub CLI (if you haven’t already)

    Authenticate with your GitHub account to use the CLI commands.

    gh auth login

    Choose GitHub.com → HTTPS → Login with browser.

  • 8. Create a new GitHub repository for your Unity project

    This links your local project to a new GitHub repo.

    gh repo create MyUnityProject --private --source=. --remote=origin # or make it public gh repo create MyUnityProject --public --source=. --remote=origin
  • 9. Rename your branch and push your code

    Rename your default branch to main and push it to GitHub.

    git branch -M main git push -u origin main
  • 10. Enable Git LFS (optional for large files)

    Use this if your project contains large assets like .psd, .fbx, or .wav files.

    git lfs install git lfs track "*.psd" "*.fbx" "*.wav" git add .gitattributes git commit -m "Configure Git LFS for large files" git push
  • 11. Verify your repository online

    Open the repo directly in your browser.

    gh repo view --web
  • 12. Make future changes and commit them

    Every time you make changes in Unity:

    git add . git commit -m "Describe what you changed" git push
  • 13. (Optional) Add collaborators

    Invite teammates to contribute to your project.

    gh repo add-collaborator username

    Or do it from your GitHub repo’s “Settings → Collaborators” page.

Notes

  • Make sure your Unity Library/ and Temp/ folders are not included — the Unity .gitignore handles this.
  • To pull your project on another computer: gh repo clone <username>/MyUnityProject
  • Use descriptive commit messages for teamwork and easier debugging.
  • Run git status anytime to see what’s changed before committing.
